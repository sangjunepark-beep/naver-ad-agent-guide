## Purpose

네이버 검색광고 AI 에이전트의 **수집→분석→제안→승인→반영** 엔드투엔드 자동 파이프라인이 멈췄을 때, 원인을 단계별로 찾아 재가동하고, Phase 4에서 이탈된 **05_실행로그 기록 경로**를 바로잡는 절차.

**사용 시점**
- 03_실_ROAS_대시보드 조회일자가 2일 이상 뒤처져 있을 때
- 04_제안기록·03_이상치로그에 어제·오늘 자동 기록이 없을 때
- Phase 4 웹훅 호출은 성공했는데 05_실행로그에 행이 추가되지 않았을 때
- 일일 9시 Cron 실행 이력이 누락될 때

## Procedure

### Step 1: n8n 실행 이력 스냅샷 확보

```
1. https://hanasignmall.app.n8n.cloud/home/executions 접속
2. 최근 7일치 실행 이력 스크린샷 저장
3. 필터: "failed" / "success" 각각 캡처
```

**Expected result:** 마지막 정상 실행 일시 확인, 실패 유형 파악 (Credential 만료 / API 스키마 변동 / 스케줄 비활성).
**If it fails:** 로그인 세션 만료 → 재로그인. SSO 붙어 있으면 구글 재인증.

---

### Step 2: 메인 파이프라인 Schedule Trigger 활성 여부 확인

```
1. "네이버 검색광고 에이전트_파이프라인" 워크플로우 진입
2. 우상단 Active 토글 확인 (ON / OFF)
3. 첫 노드 Schedule Trigger 클릭 → Rule 탭
   - 실행 시각: 매일 09:00 KST (Cron: 0 0 * * * UTC)
   - Timezone: Asia/Seoul
4. "Execute Workflow" 버튼으로 수동 1회 실행
```

**Expected result:** Active=ON, Timezone=Asia/Seoul, Cron 설정 정상. 수동 실행 시 전체 노드 초록 체크.
**If it fails:**
- Active=OFF → ON으로 토글 (원인 메모: 누가 껐는지, 언제 꺼졌는지)
- Credential 오류 → `Credentials` 메뉴에서 네이버 API Key/Secret/CustomerId 재등록
- 429/401 → Step 6 트러블슈팅 표 참고

---

### Step 3: 메인 파이프라인 수동 1회 실행 + 시트 반영 확인

```
1. 수동 실행 후 각 노드의 output 개수 확인
   - 01_캠페인_수집: 캠페인 수 ≥ 1 
   - 02_AD_리포트: 소재 수 ≥ 1
   - 02_전환_리포트: 구매전환 블록 수신
   - LLM 4단계 (Analyst→Strategist→Critic→Verifier): 각 단계 응답 생성
   - 시트 Append: 01_일일수집로그 / 02_전환수집로그 / 03_이상치로그 / 04_제안기록 각각 row 추가
2. 구글 시트에서 오늘 날짜 행이 추가됐는지 교차 확인
```

**Expected result:** 4개 시트에 오늘자 행이 1건 이상 신규 기록. 03_실_ROAS_대시보드의 조회일자 셀이 오늘로 갱신.
**If it fails:** 어느 노드에서 멈췄는지 기록 → Step 6 참조.

---

### Step 4: Phase4_승인_자동반영 워크플로우 점검

```
1. "Phase4_승인_자동반영" 워크플로우 진입
2. Webhook 노드 URL 복사 (Production URL)
3. 마지막 정상 실행(#133) 열기 → 16개 노드 구조 재확인
4. Google Sheets "Append Row" 노드 클릭
   - Document ID: 1Yuw_8we4nEzL1nslHI66LHBBE_uWc-ErALzhn2vvLGI
   - Sheet (gid/이름): 반드시 "05_실행로그" 지정 확인
   - Mapping Mode: Map Each Column Manually
   - 필드 매핑: 실행ID / 제안ID / 실행일시 / 키워드 / 변경전 / 변경후 / 실행결과 / 승인자 / 사후ROAS / 효과 / 메모
5. 테스트: 이 노드 단독 "Execute Node" → 시트에 테스트 행 1개 추가되는지 확인
```

**Expected result:** 시트명 "05_실행로그" 정확히 연결. 단독 실행으로 05_실행로그에 테스트 행 신규 생성.
**If it fails:**
- 시트명이 "시트1" "Sheet1" 또는 공란 → 드롭다운에서 "05_실행로그" 재선택
- 컬럼 매핑 깨짐 → 헤더 행(Row 3) 기준으로 재매핑
- "The caller does not have permission" → 구글 OAuth Credential 재인증

---

### Step 5: Phase 4 엔드투엔드 재테스트 (소폭 ±1%)

```
1. 04_제안기록에 임시 테스트 행 1건 생성 (예: P999, 현재 입찰가 기준 -1% 제안)
2. Webhook URL에 POST (curl 또는 n8n 내 "Respond to Webhook" 테스트)
   - Body: {"제안ID":"P999","승인":"Y","승인자":"박상준"}
3. 네이버 검색광고 API로 해당 소재 입찰가 변경 확인 (GET)
4. 05_실행로그에 E00X 신규 행 자동 추가 확인
5. 변경 후 10분 내 동일 값으로 원복 (수동)
```

**Expected result:** 3곳 모두 반영 (네이버 API 입찰가 변경 / 05_실행로그 행 추가 / 04_제안기록 상태 컬럼 "실행완료").
**If it fails:** Step 6 트러블슈팅 참고.

## Troubleshooting

| Symptom | Likely Cause | Fix |
|---------|-------------|-----|
| 오늘 실행 이력 0건 | Active=OFF 또는 Timezone UTC로 변경됨 | Step 2에서 Active ON + Timezone Asia/Seoul 재설정 |
| 401 Unauthorized (네이버 API) | API Secret 재발급 또는 IP 차단 | 광고 관리자 → API 관리 → Secret 재발급 + n8n Credential 교체 |
| 308 Permanent Redirect | base URL 오타 (`api.naver.com` 사용) | `https://api.searchad.naver.com`으로 교체 |
| 429 Too Many Requests | LLM/네이버 API 호출 과다 | Split In Batches 노드 추가 또는 Wait 노드 5초 삽입 |
| Google Sheets "permission denied" | OAuth 토큰 만료 | n8n Credential 재인증 (Google OAuth2) |
| 05_실행로그에 빈 행만 추가됨 | Append 노드 매핑이 컬럼번호 기준인데 시트 구조 변경됨 | Map Each Column Manually로 교체 + 헤더 이름 기준 재매핑 |
| Analyst 노드 "Invalid JSON" | LLM 응답에 백틱/주석 섞임 | System prompt에 "Return JSON only, no markdown" 명시 |
| Critic이 Strategist를 계속 반박 → 무한루프 | LLM 단계별 temperature 동일 | Verifier에 max iteration = 2 가드 추가 |
| 입찰가 10원 단위 아님 (HTTP 400) | 반올림 로직 누락 | Function 노드에서 `Math.round(bidAmt/10)*10` |
| 자사몰 매출 0인데 실_ROAS 0% | CS/전화 주문 미반영 (구조적 제약) | 메모만 남기고 판정 보류 (memory: hanasignmall CS offline 참고) |

## Rollback

**변경한 것만 역순으로 되돌릴 것.**

| 변경 대상 | 복원 방법 |
|-----------|-----------|
| n8n 워크플로우 JSON | `네이버_검색광고_에이전트_파이프라인.json` / `Phase4_승인_자동반영.json` 백업본 재Import |
| 입찰가 실 변경 | 네이버 광고 관리자에서 이전 값으로 수동 원복 (editTm으로 변경 전 값 역추정) |
| 05_실행로그 잘못 기록된 테스트 행 | 해당 행 우클릭 → 행 삭제 (단, 실제 API 호출이 나간 경우 기록 보존) |
| Credential 교체 실패 | 이전 Credential을 Disable 상태로 유지 (삭제 금지) → 롤백 시 재활성 |

**원복 기준**: 수동 실행 한 번으로 파이프라인이 정상 기록을 남기지 못하면, 30분 내 원복 후 재분석.
