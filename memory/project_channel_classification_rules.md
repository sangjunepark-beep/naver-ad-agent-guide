**규칙** (채널별_집계 노드 mapChannel 함수):

```js
if (s.includes('(영호)')) return '영업(영호)';
if (s === '스마트스토어') return '스마트스토어';
if (s === '고도몰5') return '자사몰(com)';
if (s === '신규몰') return '자사몰(kr)';
if (s === '쿠팡(신)') return '쿠팡';
if (s === 'G마켓') return 'G마켓';
if (s.startsWith('사인몰') || s.startsWith('하나몰')) return 'CS';
return '기타';
```

매칭 예시:
- `사인몰(영호)` / `사인몰(영호)(현주)` / `사인몰(영호)(민주)` / `하나몰(영호)(태경)` → 영업
- `사인몰(정란)` / `사인몰(민주)` / `사인몰(현정)(태경)` / `하나몰(현정)(태경)` → CS
- `네이버페이`, `11번가`, `옥션` → 기타 (의도된 분류. 필요 시 추가)

**Why**: 2026-04-24 확인 — 이전 코드는 `=== '사인몰(영호)'` 정확 일치만 영업으로 잡아서 `사인몰(영호)(X)` 복합 진행자가 전부 CS로 흘러감. 04-23 기준 영업 12건/4,679,950원이 CS에 섞여있고 영업 표시는 0이었음. 또 `하나몰(X)`은 `startsWith('사인몰')`에 안 걸려 '기타'로 빠짐 (04-23 하나몰 2건/854,500원).

**How to apply**:
- 이 규칙대로 수정 후 04-23 CS 21건/5,804,460원, 영업 12건/4,679,950원이 분리 표시됨 (일자 보정 적용 시 CS 22건/6,033,460원)
- 과거 리포트의 CS 매출은 영업+하나몰 포함 과대값이었으니 주간/월간 평균 비교 시 바이어스 고려 필요
- 신규 진행자 명칭 등장 시 mapChannel에 추가해야 '기타'로 빠지지 않음 — exec 결과 `buckets.기타.건수 > 0`이면 감지 가능
