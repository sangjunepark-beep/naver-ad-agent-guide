**정확한 정책 (박상준 차장 확인 2026-04-20)**
- 파이프라인 전체(수집·분석·사후 ROAS 측정): **매일 가동** (주말·공휴일 포함)
- 리포트 메일 발송만: **영업일(월~금 + 공휴일 제외)** 한정

**현재 구현 (잘못된 상태)**
```
스케쥴러 cron: "0 9 * * 1-5"   ← 월~금만 09:00 트리거 (주말 skip)
    ↓
공휴일_확인 → 공휴일_건너뜀 (IF)   ← 공휴일이면 파이프라인 전체 중단
    ↓
변수_설정 → 수집 → 분석 → Gmail
```

**Gap**
- 주말 이틀치 데이터 **매주 누락** 중
- 공휴일 데이터도 누락
- Phase 5-1 사후 ROAS 측정(변경일 -3일~-1일 비교)에 구멍 발생

**수정안 (미착수)**
1. 스케쥴러 cron: `0 9 * * 1-5` → `0 9 * * *` (매일)
2. `공휴일_확인` 로직에 주말(dow 0·6) 판정 추가
3. `공휴일_건너뜀` IF 노드를 파이프라인 시작부 → **Gmail_발송 직전**으로 이동
4. 메일만 영업일 차단, 수집·분석·사후 ROAS는 매일 수행
- 예상 작업 시간: 30~40분

**How to apply**: Phase 5 선행 과제에 추가. R3/R4와 함께 이번 주~다음 주 중 처리. MEMORY.md 인덱스와 `project_overall_progress.md` Phase 5-4 설명도 같이 정정 필요(본 메모가 정정본 근거).
