**사실** (2026-04-24 재검증): 04-14 메모 `project_naver_ad_conversion_inflation.md`의 "네이버 총 전환매출에 장바구니 가치 포함되어 실매출 5배 부풀림" 주장은 **과장**이었음. 실제는:

1. **네이버 Conv API는 purchase·add_to_cart·wishlist 등을 `convType` 필드로 명확히 분리해서 제공**
2. **n8n `Conv_TSV_파싱` 노드는 convType=1(purchase)만 `구매전환매출`에 합산, 장바구니는 `장바구니추정매출` 별도 컬럼에 저장** — 설계 정상
3. 04-23 원본 Conv TSV 분석 결과: 총 27행, purchase 13건/1,087,500원(리포트 기여매출과 일치) + cart 22건/3,439,910원(별도)
4. **같은 소재에 purchase 행이 중복된 경우 0건** — 공식 last-click 어트리뷰션대로 동작 확인

**실결제 대비 부풀림 규모** (04-13~22 누적):
- n8n 구매전환매출: 29,501,900원
- 업체계약현황 스마트스토어: 15,818,900원 → 1.86배
- 스마트스토어+신규몰+네이버페이+고도몰5 합산 (네이버 광고 유입 가능 채널 상한): 18,738,500원 → **1.57배**
- 자사몰(04-23 등록까지 포함): 20,089,300원 → **1.47배**

**잔여 부풀림 원인 후보** (중복 계상 아님):
- 상품 단위 매출 합산 (1 주문 N 상품이면 N 행 × 상품별 매출)
- 주문 취소·환불 미반영 (Conv API는 결제 시점 집계, 이후 취소 차감 없음)
- 간접전환 20일 창 멀티터치 어트리뷰션(공식 설정에 따라)
- 할인 전/후 가격 집계 기준 (2019-05-08 이후 결제가 기준, 그러나 여전히 업체계약현황과 격차 존재)

**공식 문서 근거**:
- 네이버 스마트스토어/브랜드스토어 자동연동: **"장바구니 전환매출액은 리포트에 0원으로 반영"** (2022-09-22 공지)
- wcs.trans 신 스크립트: `_conv.value`는 purchase의 전체 거래금액, 총 전환매출 = purchase value 합산
- 임대몰(카페24 2024-07~, 고도몰5 2024-12~)은 개별 이벤트 해제 기능 미제공이지만 cart는 매출 0원으로 찍힘

**How to apply**:
- **04-14 메모의 "5배 부풀림, 장바구니 value 합산" 주장은 철회**. 대신 1.3~1.9배 수준의 구조적 차이 인정하고 해석 시 반영
- 리포트 수치는 그대로 쓰되, **"네이버 집계 기준(실결제 대비 약 1.5배 과대 가능)"** 주석 병기 권장
- 실결제 기반 ROAS가 꼭 필요하면 **스마트스토어 판매자센터 주문 API 연동**(장기)이 유일한 근본해법. 중간 단계로 업체계약현황 시트 기반 일별 보정계수 적용 가능(H-L1 방안)
- **장바구니는 이탈률·관심도 관찰 지표로 활용**. 리포트에 장바구니 섹션 별도 추가(F' 작업)하면 purchase/cart 명확히 구분된 대시보드 완성
- 차장님 이해: "한 소재에서 타고 들어와 바로 구매(직접)와 돌다가 다시 와서 구매(간접)"는 네이버가 별도 컬럼(`직접구매전환매출`/`간접구매전환매출`)으로 정확히 구분해줌. 중복 계상 아님
